# Agromonitoring (Система мониторинга сельхозформирований)

Agromonitoring — это информационно-аналитическая система для мониторинга применения удобрений и пестицидов в аграрном секторе. Проект состоит из разделенных клиентского приложения (Single Page Application) и REST API сервера.

---

## Архитектура проекта

Проект представляет собой монорепозиторий, разделенный на две независимые части:
- `/frontend` — Клиентское приложение (React + Vite)
- `/backend` — Серверная часть API (Java + Spring Boot + PostgreSQL)

Данная архитектура позволяет вести независимую разработку, свободно масштабировать каждую часть и использовать контейнеризацию для деплоя.

---

## Стек технологий

### Клиентская часть (Frontend)
- **Ядро:** React 18, Vite (для быстрой сборки и HMR)
- **Маршрутизация:** React Router v6
- **Интерфейс:** Ant Design (UI-компоненты: формы, таблицы, модальные окна)
- **Визуализация:** Recharts (графики) и Leaflet / React-Leaflet (карты)
- **Работа с сетью:** Axios с настроенными перехватчиками (interceptors) для обработки ошибок и авторизации
- **Документооборот:** jspdf, docx, xlsx

### Серверная часть (Backend)
- **Платформа:** Java 21, Spring Boot 4.0.5
- **База данных:** PostgreSQL 14+ (ORM через Spring Data JPA / Hibernate)
- **Безопасность:** Spring Security + JWT Tokens
- **Сборка:** Gradle
- **Утилиты:** Lombok (для уменьшения рутинного кода)

## Текущий статус разработки (Что реально реализовано)

Система находится в процессе активной разработки. На данный момент фронтенд-часть значительно опережает бэкенд по количеству интерфейсов. 

### Реализовано на стороне Frontend
- Полностью готовые интерфейсы: страницы входа, дашбордов, реестров и калькуляторов.
- Настроенная безопасность: защищенные роуты, проверка ролей (Admin, Staff, User).
- Связка с реальным бэкендом: авторизация (JWT) и справочник продуктов.
- Мок-данные: все остальные страницы (калькулятор, дашборд аналитики, отчеты, справочники) временно завязаны на локальный `json-server`, пока на стороне бэкенда не будут реализованы соответствующие контроллеры.

### Реализовано на стороне Backend
- **Авторизация (AuthController):** Полностью рабочий цикл логина, регистрации и логаута с проверкой паролей через BCrypt.
- **Безопасность (JwtAuthenticationFilter):** Настроена защита эндпоинтов, извлечение токенов и ролей, настроен CORS для портов локальной разработки фронтенда.
- **Продукты (ProductController):** Полный CRUD-цикл управления продуктами через базу данных.
- **Заглушки:** Заявленные классы `CalculatorController`, `StatsController` и логика отчетов пока существуют в виде пустых файлов-заглушек в ожидании реализации бизнес-логики.

## Схема маршрутизации запросов

Фронтенд в данный момент обращается к двум разным источникам:
1. Запросы к **API авторизации и управления продуктами** уходят по относительному пути `/api/v1/` и проксируются средствами сборщика Vite на Java-бэкенд (на порт 8080).
2. Запросы к **дашбордам, калькуляторам и отчетам** используют абсолютный путь (указан в `.env` как `VITE_API_BASE_URL`) и обращаются напрямую к временному JSON-серверу (на порт 3001).

По мере готовности серверных контроллеров, обращения к JSON-серверу будут переведены на реальный бэкенд.

```text
agroproject/
├── backend/                  # Серверная часть Spring Boot
│   ├── build.gradle.kts      # Конфигурация сборки и зависимости
│   └── src/main/java/agro_back/demo/
│       ├── config/           # Настройки Spring Security и CORS
│       ├── controller/       # Контроллеры REST API
│       ├── dto/              # Объекты передачи данных (AuthRequest и т.д.)
│       ├── entity/           # Сущности базы данных (User, Product)
│       ├── repository/       # Интерфейсы Spring Data JPA
│       ├── security/         # Утилиты JWT и фильтры запросов
│       └── service/          # Изолированная бизнес-логика
│
└── frontend/                 # Клиентская часть React
    ├── db.json               # Файл с мок-данными для временного сервера
    ├── package.json          # Зависимости проекта Node.js
    ├── vite.config.js        # Настройки сборщика и прокси-сервера
    └── src/
        ├── api/              # Абстрактный слой работы с сетью (клиенты HTTP)
        ├── components/       # Общие структуры UI (Header, Layout)
        ├── context/          # React Context (Глобальное состояние авторизации)
        ├── pages/            # View-компоненты экранов системы
        └── router/           # Конфигуратор маршрутов
```

## Инструкция по локальному запуску

Для полноценной работы системы в режиме разработки необходимо запустить базу данных, бэкенд, сервер фиктивных данных и сам фронтенд. Рекомендуется использовать три отдельных окна терминала.

### 1. Подготовка базы данных
Убедитесь, что установлен и запущен PostgreSQL. Создайте базу данных с названием `agro_db`:
```sql
CREATE DATABASE agro_db;
```
Если ваши учетные данные отличаются от `postgres`, обновите их в файле `backend/src/main/resources/application.properties`.

### 2. Запуск Backend (Java Spring Boot)
В первом окне терминала перейдите в папку бэкенда и запустите приложение. Убедитесь, что установлена Java 21:
```bash
cd backend
./gradlew bootRun
```
Приложение запустится на порту `8080`. База данных инициализируется автоматически.

### 3. Запуск Mock Server
Во втором окне терминала перейдите в папку фронтенда, установите зависимости и запустите JSON-сервер для поддержки тех интерфейсов, которые еще не реализованы на бэкенде:
```bash
cd frontend
npm install
npm run server
```
Сервер запустится на порту `3001`.

### 4. Запуск Frontend
В третьем терминале запустите сервер разработки Vite:
```bash
cd frontend
npm run dev
```
Клиентское приложение запустится локально (по умолчанию на порту `5173`). Откройте предоставленную в консоли ссылку в браузере.
